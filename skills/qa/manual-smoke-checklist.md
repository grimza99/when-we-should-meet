# 수동 스모크 체크리스트

이 체크리스트는 자동화 테스트가 이미 커버하는 범위를 다시 반복하기 위한 문서가 아니다.
사람이 실제 브라우저, 실제 디바이스, 실제 외부 연동 환경에서만 확인할 수 있는
항목을 빠르게 점검하기 위한 수동 스모크 체크리스트다.

사용 시점:

- 기능 PR을 `ready for review`로 올리기 직전
- 데모 빌드를 확인할 때
- 배포 직전 최종 점검할 때

이 앱은 모바일 폭 전용이므로, 모바일 viewport 또는 실제 모바일 디바이스 기준으로 확인한다.

## 준비

- 배포 도메인 또는 실제 Firebase가 연결된 환경을 준비한다.
- iPhone Safari, Android Chrome 중 최소 1개 이상 실제 디바이스를 준비한다.
- 가능하면 브라우저 프로필 2개 또는 디바이스 2대를 준비해 멀티 참가자 흐름을 확인한다.
- 카카오 공유 확인이 필요하면 카카오톡이 설치된 모바일 디바이스를 준비한다.
- 자동화 테스트 결과가 이미 통과했는지 먼저 확인한다.
  - `npm run test:unit`
  - `npm run test:e2e:local`
  - `npm run test:e2e:kakao`
  - `npm run test:e2e:firebase`

## 핵심 스모크

- 랜딩 페이지가 모바일 폭에서 깨지지 않고 첫 화면에서 핵심 CTA가 즉시 보인다.
- 방 생성부터 닉네임 입력, 달력 진입까지 한 번의 happy path가 실제 환경에서 자연스럽게 이어진다.
- 공유된 방 링크를 새 브라우저 또는 다른 디바이스에서 열었을 때, 부자연스러운 깜빡임이나 잘못된 초기 화면이 보이지 않는다.

## 실기기 및 브라우저

- iPhone Safari 또는 Android Chrome에서 버튼 터치 영역이 충분하고, 스크롤/입력/모달 진입이 불편하지 않다.
- 모바일 키보드가 열렸을 때 닉네임 입력, 초대코드 입력, 날짜 범위 입력 UI가 가려지지 않는다.
- 화면 전환, 모달 열기/닫기, 토스트 노출이 실제 기기에서 어색하거나 밀리지 않는다.
- 공유 후 돌아왔을 때 현재 방 상태가 유지된다.

## 실시간 멀티 참가자

- 실제 Firebase가 연결된 환경에서 두 클라이언트가 같은 방을 열고 있을 때, 한쪽 변경이 다른 쪽에 눈에 띄게 반영된다.
- 참가자 제거, 참가자 나가기, 방 삭제처럼 상태 변화가 큰 액션 이후에도 다른 클라이언트가 이해 가능한 화면 상태로 이동한다.
- 새로고침 또는 앱 전환 후 복원이 실제 사용감 기준으로 어색하지 않다.

## 카카오 공유

- 카카오 공유 버튼을 누르면 실제 카카오 공유 UI 또는 앱 전환이 발생한다.
- 카카오 공유 후 복귀했을 때 앱이 깨지지 않고 현재 방 화면을 유지한다.
- 실제로 전달된 공유 카드/문구/링크가 기대한 목적에 맞는지 확인한다.

## 배포 도메인 및 링크

- 배포 도메인 기준 공유 링크를 복사해 다른 브라우저 또는 다른 디바이스에서 열 수 있다.
- 링크 진입 후 방 참여, 방 조회, 접근 제한 화면이 환경에 따라 다르게 깨지지 않는다.
- 브라우저 주소창, OS 공유 시트, 카카오 공유 등에서 링크가 이상한 도메인이나 로컬 주소를 가리키지 않는다.

## 시각 및 문구

- 랜딩 hero, 빈 상태, 오류 토스트, full-room 메시지, 접근 제한 메시지가 실제 사용자 기준으로 자연스럽다.
- 달력 dot, 랭킹 badge, 참가자 리스트가 실제 화면에서 겹치거나 잘리지 않는다.
- 다크모드, 확대 텍스트, 브라우저 기본 폰트 차이로 인해 레이아웃이 크게 무너지지 않는다.

## 이번 체크리스트에서 제외하는 것

아래 항목은 자동화 테스트가 이미 소유하므로, 매번 수동 스모크에서 반복 확인하지 않는다.

- 초대코드 정규화와 빈값 차단
- 방 생성 폼의 세부 유효성 검사
- 닉네임 빈값 차단
- 캘린더 선택/해제, 요일 규칙, 랭킹 계산의 상세 규칙
- Firestore 문서 생성/삭제 자체의 기술적 검증
- listener resilience와 같은 fault-injection 시나리오
- `build`, `lint`, PR 대상 브랜치, 리뷰 코멘트 존재 여부 같은 프로세스 체크

## 사인오프 메모

- 어떤 디바이스와 브라우저에서 확인했는지 기록한다.
- 카카오 실제 공유 UI 확인 여부를 기록한다.
- 수동으로만 발견된 이슈가 있으면 PR 또는 테스트 플랜 문서에 남긴다.
