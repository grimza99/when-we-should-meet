# 업무일지 문서 자동화 에이전트 브리핑

이 문서는 `when-should-we-meet` 프로젝트에서
업무일지 정리 자동화를 담당하는 에이전트용 기준 문서다.

<strong>버전 : </strong> v2

<strong>생성 날짜 : </strong> 2026-04-15

<strong>최신 업데이트 날짜 : </strong> 2026-05-19

## 목적

- 에이전트와 사용자의 맥락 동기화를 위한 문서로, 사용자가 해당 일자의 업무 내용 파악을 알기 쉽게 도와주는 문서이다.

## 기본 규칙

- 사용자가 Notion 페이지를 요청하면, 별도 지시가 없는 한 프로젝트에서 합의된
  루트 아래 하위 페이지로 생성한다.
- 업무일지는 날짜 제목 `yyyy-mm-dd` 형식을 기본으로 사용한다.
- 각 에이전트를 구별하고, 에이전트의 하위 내용에 작업에 대한 내용을 적는다.

## 문서 품질 기준

- 사용자의 이해도를 최우선으로 공백, 줄바꿈, 들여쓰기를 활용하여 가독성을 높인 문서를 만든다.
- 사용자의 이해도를 높이기 위해 두괄식으로 서술한다.

## 기본 형식

### 에이전트명

- QA , 메인 에이전트, 배포담당, 마케팅 등 하위 에이전트의 역할 명시

#### task type에 따른 보고

- 아래 task type 에 따른 형식을 참고하여, notion의 code block 을 활용하여 각 task type에 맞게 리스트 업 한다.

#### 이후 착수 계획

- 이후 진행 예정에 대한 내용을 작성한다.

## task type 형식

### task type1 - 개발후 PR 생성 완료

`[DONE] n
branch : 브랜치 명
pr : pr 링크
merge : 병합 여부
내용 : 해당 PR에서 구현한 내용
기타 : 기타 내용`

### task type2 - 개발후 PR 생성 미완료

`[DONE] n
branch : 브랜치 명
내용 : 해당 브랜치에서 구현한 내용
기타 : e.g.) 커밋은 완료하지 않았습니다 or 커밋 완료후 워킹트리는 클린 상태입니다 `

### task type3 - 개발외 업무

`[DONE] n
개요 : 업무 내용
기타 : e.g.) 커밋은 완료하지 않았습니다 or 커밋 완료후 워킹트리는 클린 상태입니다 
`
